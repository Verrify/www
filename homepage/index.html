Production is Chaotic

Develop Resilience to the Unpredictable with Verrify's Conditional Chaos

  • codetags Create custom app conditions to easily cover input and configuration possibilities.
  • phonelink Test against device conditions such as low free space and feature access.
  • perm_scan_wifi Control network conditions from bandwidth to loss, lag, and network changes.
  • codetags Reproduce service conditions like server downtime, unexpected HTTP response codes, or missing response data.
  • done_all Verrify is preloaded with common scenarios to jump start your testing.


Verrify easily integrates into your existing test suite. You can also use Verrify with no code changes required via the Verrify app.

			                   
/* Example: Annotations for custom conditions */ 
#include <verrify.h>

/* Function is used by a sync engine to save files. */
int write_file(FILE *fd) {
  /* Annotate the function with verrify_return_on */
  verrify_return_on("no_free_space", -1);

  /* [...] */ 
}

/* What happens to existing files if the sync operation fails? */
/* They should be present & unmodified. Is that what happens?  */
int test_sync(void) {
  /* Given some defined test data */  
  /* Enable no_free_space to cause write_file to return -1 */
  ex_set_test_data(); 
  verrify_enable("no_free_space");
  
  /* Initiate a sync and then confirm the data */ 
  /* still matches the preloaded test data */
  sync();
  assert(current_data_matches_test_data());
}
